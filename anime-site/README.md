# QIK Anime — фронтенд 🎬

Минималистичный аниме-сайт на **React + Vite** с пастельным акцентом (лавандово-мятный градиент).

Контент берётся из [YummyAnime API](https://old.yummyani.me/api/swagger), а аккаунты,
закладки, рейтинги и комментарии — из собственного **NestJS-бэкенда** (см. `../server`).

## ✨ Возможности

- **Главная** — герой-карусель трендов, новые серии, свежие релизы, анонсы.
- **Каталог** — фильтры по жанрам / типу / статусу / сортировке + подгрузка.
- **Страница аниме** — постер, статистика, описание, похожие тайтлы,
  **закладки**, **пользовательский рейтинг** и **комментарии**.
- **Просмотр** — встроенный плеер с выбором озвучки, плеера и эпизодов.
- **Расписание** — выход онгоингов по дням недели.
- **Поиск** + **Закладки** (личная библиотека по статусам просмотра).
- **Аккаунты** — регистрация/вход в модальном окне, JWT-сессия.

## 🚀 Запуск

```bash
npm install
npm run dev      # http://localhost:5173
npm run build
npm run preview
```

> Для работы аккаунтов/закладок/оценок/комментариев нужен запущенный бэкенд (`../server`).

## ⚙️ Переменные окружения

```bash
cp .env.example .env
```

| Переменная | Назначение | По умолчанию |
|---|---|---|
| `VITE_QIK_API_URL` | URL NestJS-бэкенда | `http://localhost:3001/api` |
| `VITE_YUMMY_APP_TOKEN` | опциональный токен YummyAnime | — |

## 🛠 Стек

React 18 · React Router 6 · Vite 5 · чистый CSS (дизайн-система на CSS-переменных).

## 📁 Структура

```
src/
├── api/
│   ├── client.js        # YummyAnime API (каталог, поиск, видео)
│   └── backend.js       # QIK backend (auth, закладки, рейтинг, комменты)
├── context/AuthContext  # сессия, модалка входа, тосты
├── components/          # Header, AuthModal, BookmarkButton, RatingWidget, Comments …
├── pages/               # Home, Catalog, AnimeDetail, Watch, Schedule, Search, Library
└── styles/index.css     # дизайн-система (пастельная тема)
```
