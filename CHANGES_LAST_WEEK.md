# Изменения за неделю (17–24 июня 2026)

Всего 7 коммитов. Основные темы: рейтинговая система, профиль пользователя, постеры.

---

## 1. Рейтинги опенингов/эндингов + страница «Рейтинги»

**fa742eb** — самая крупная фича недели (13 файлов, +567/-26 строк).

- Новая сущность `OpeningRating` — оценка OP/ED по шкале 1–10, хранится в таблице `opening_ratings` с составным уникальным индексом `(user, animeId, type)`.
- Компонент `OpeningRatingWidget` — звёздочки на странице аниме для оценки опенинга и эндинга отдельно.
- Новая страница `/ratings` с четырьмя вкладками-лидербордами:
  - **Аниме** — топ по средней оценке (≥3 оценок)
  - **Опенинги** — топ OP по средней оценке (≥2 оценок)
  - **Эндинги** — топ ED по средней оценке (≥2 оценок)
  - **Пользователи** — топ по уровню (XP из всей активности)
- Ссылка «Рейтинги» добавлена в выпадающее меню аватара в Header.
- Прокси-контроллер постеров доработан: явный `Cache-Control` через декоратор `@Header`, `passthrough: true` для Response.
- Бэкенд: новые эндпоинты в `ratings.controller.ts` — `POST /ratings/opening`, `GET /ratings/opening/:animeId`, `DELETE /ratings/opening/:animeId`, `GET /ratings/top/anime`, `GET /ratings/top/openings`, `GET /ratings/top/endings`, `GET /ratings/top/users`.

## 2. Полный XP в топе пользователей

**63d2c59** — 1 файл, +50/-5 строк.

- Расчёт XP в `topUsers()` теперь совпадает с профилем (`computeXp()`).
- Добавлен учёт закладок со всеми статусами: `completed` (+8 XP), `watching` (+5 XP), `rewatching` (+5 XP), `planned` (+2 XP), `on_hold` (+2 XP), `favorite` (+3 XP).
- Extra-эпизоды из закладок с вычетом уже просмотренного (`watch_progress`) — +10 XP за эпизод. Для статуса `watching` берётся 50% эпизодов (минимум 1).

## 4. Мастера могут редактировать и удалять любые комментарии

**af25dd8** — 2 файла, +4/-2 строк.

- Раньше мастера могли только удалять чужие комментарии. Теперь могут и редактировать.
- На фронте: условие `canModerate` в `Comments.jsx` теперь разрешает и `edit`, и `delete`.
- На бэкенде: `CommentsService.update()` теперь проверяет `isMaster` наравне с `isAdmin`.
